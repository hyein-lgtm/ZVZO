# ZVZO 판매 비서 — 설치 & 배포 가이드

ZVZO 벤더 관리자에 자동 로그인해서 **판매현황·셀러일정·상품** 화면을 긁어오고,
당신이 자연어로 물어보면 Claude가 정리해 답해주는 작은 웹 앱입니다.

---

## 0. 준비물 (3가지)

1. **GitHub 계정** (코드를 올릴 곳)
2. **Railway 계정** (앱을 돌릴 곳) — https://railway.app
3. **Claude(Anthropic) API 키** — https://console.anthropic.com → API Keys 에서 발급

---

## 1. 내가 직접 채워야 하는 정보 4개

`.env.example` 파일을 열어 아래 값을 준비하세요. (코드는 안 건드려도 됩니다)

| 항목 | 무엇 |
|---|---|
| `ANTHROPIC_API_KEY` | 위에서 발급한 Claude API 키 |
| `ZVZO_LOGIN_URL` | ZVZO 로그인 페이지 주소 |
| `ZVZO_USERNAME` | ZVZO 로그인 아이디 |
| `ZVZO_PASSWORD` | ZVZO 로그인 비밀번호 |

> **로그인 주소를 모르면?** ZVZO에서 로그아웃했다가 다시 로그인할 때
> 주소창에 뜨는 주소가 `ZVZO_LOGIN_URL`입니다. (예: `https://store.zvzo.shop/login`)

> **셀러일정·상품 화면도 보고 싶으면?** 그 화면을 연 상태에서 주소창의 주소를 복사해
> `scraper.py`의 `TARGET_PAGES`에 주석을 풀고 붙여 넣으세요.

---

## 2. GitHub에 올리기

1. GitHub에서 새 저장소(Repository)를 만든다 (예: `zvzo-agent`, Private 권장)
2. 이 폴더의 파일들을 그 저장소에 업로드
   - 웹에서 드래그&드롭으로 올려도 됩니다
   - ⚠️ 비밀번호가 든 `.env` 파일은 **절대 올리지 마세요** (`.env.example`만 올림)

---

## 3. Railway에 배포하기

1. Railway 접속 → **New Project → Deploy from GitHub repo** → 방금 만든 저장소 선택
2. Railway가 `Dockerfile`을 자동 인식해 빌드를 시작합니다 (3~5분 소요)
3. 빌드 중에 **Variables(환경변수)** 탭으로 가서 `.env.example`의 5개 값을 입력
   - 키 이름은 그대로, 값만 본인 것으로
4. **Settings → Networking → Generate Domain** 클릭 → 공개 주소(URL) 생성
5. 그 주소로 접속하면 질문 입력창이 뜹니다 🎉

---

## 4. 사용하기

질문 예시:
- "오늘 판매 현황 한 줄로 요약해줘"
- "주문 건수랑 매출 알려줘"
- "이번 주 셀러 일정 표로 정리해줘"
- "재고 적은 상품 있어?"

처음 질문이나 '새로고침' 체크 시 화면을 긁느라 10~20초 걸립니다. (이후 5분간은 빠름)

---

## 5. 잘 안 될 때 (자주 있는 문제)

**① "로그인이 안 되는 것 같다 / 빈 화면만 나온다"**
→ 로그인 입력칸 모양이 특이한 경우입니다. 로그인 페이지에서 아이디 칸을
   마우스 오른쪽 클릭 → **검사(Inspect)** → 파란색으로 표시된 `<input ...>` 안의
   `name="..."` 값을 확인해, `scraper.py`의 `_try_fill(...USERNAME)` 후보에 추가하세요.

**② "데이터가 비어 있다"**
→ `scraper.py`의 `page.wait_for_timeout(2000)` 숫자를 `5000`(5초)으로 늘려보세요.
   화면이 느리게 그려지는 경우입니다.

**③ "Claude가 엉뚱하게 답한다"**
→ 긁힌 텍스트에 메뉴·광고가 많이 섞인 경우입니다. 질문을 더 구체적으로 하거나,
   원하면 나중에 특정 영역만 긁도록 셀렉터를 좁힐 수 있습니다.

---

## 6. 알아둘 점 (솔직하게)

- ZVZO가 화면 디자인을 크게 바꾸면 긁기가 멈출 수 있어요. 그때는 주소/대기시간만 조정하면 대부분 복구됩니다.
- 자동 로그인이 ZVZO 이용약관에 어긋나지 않는지 한 번 확인하세요. (내 계정·내 데이터를 보는 용도라 보통 문제없지만, 안전하게)
- 비밀번호는 코드가 아니라 **Railway 환경변수**에만 넣습니다. 코드/깃허브에는 절대 적지 마세요.
